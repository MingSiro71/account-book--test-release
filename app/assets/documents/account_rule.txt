原則：1つのレコードに対してバックレコードは1行
例えば源泉徴収などの場合、売上と別途で源泉徴収としてレコードを作るように誘導する

表記：借方 / 貸方

売上
*このアプリ上では税込金額での入力を求めます
*当月入金の場合
  事業主貸 + / 売上 +
*掛売りの場合
  売掛金 + / 売上 +
売掛金受領
  事業主貸 + / 売掛金 + 
被源泉徴収
*利用者に手取り額と別途で入力させる
*この時、源泉徴収側には仮受消費税を発生させない＝税込から源泉徴収というロジックを採用する
*もし、課税事業者で税抜から源泉徴収のロジックを採用したい場合はロジック修正（源泉）を行って
*仮受消費税を売上に変換、サポートウィンドウを出して元の売上から算出できるようにする
  仮払税金 + / 売上 + 
消費税
*入力ごとには計算せず、決算時にみなし仕入率が使える/使うか、また課税事業者かを選択する
*決算時処理
  租税公課 + / 未払消費税 +
*納税時
  未払消費税 + / 事業主借 +

一般費用
  費用 + / 事業主借 +

事業主貸借調整(決算時に自動計算)
*事業主貸 > 事業主借 の場合
  事業主貸 = 事業主貸 - 事業主借
  事業主借 = 0
*逆の場合
  逆
